The group management list shows every group that has been added to the system . The list contains four columns.
1. Group Name The unique name of the group is shown here.
2. Members: This shows No. of grouped HyperSPROUT™.
3. Description: This column shows a brief description of what the group is for.
4. Action: This column contains a button for downloading the configurations associated to each of the group’s members and a deletion button for removing the group.

If the list has a large amount of data, it might separate into pages. In this case, the user can choose how many entries to display or which page to show from the pagination tool bar in the bottom left of the report.

To sort the list, the user must click the header of whichever column they would like to sort by. For instance, clicking on the Group Name header will switch the column between the default display, ascending alphabetically, and descending alphabetically.

The report gives four different options for exporting. The user may export either all the data in the report or just the currently displayed data in either a CSV format or a PDF format. To do this the user simply has to click on the export button in the top right corner of the report after filtering it for the desired data.

An application group is a special grouping of devices. It allowes a user to group like devices that might have different configurations. For instance, user might want to group all of the devices in a region together so that configuration updates can be sent out at the same time across the region. To create one of these groups simply click the new button above the right corner of the group list.

This will open up a creation window. Input a unique name and a brief description of the application group’s purpose then click save. A new application group matching the inputed information will immediately be added to the list.

To view basic details of a group, find the group’s entry in the group list and click on the group name. A window will appear with the group’s details.

To delete a group simply find it in the list and click the deletion button. User will be prompted if user are sure. Click yes and the group will be removed from the system (Note, the group must not have any members for the deletion button to be available).

When a group is first created it has no members associated to it. To add members locate the Assign Endpoint Group Membership button above the top right corner of the list and click it.

After clicking the button the Assign Endpoint Group Membership window will pop up. From here select a group from the dropdown (Note, user can jump to the application group creation window by clicking the plus button next to the group dropdown). Then select whether to add or remove members and input the path of the file containing the device IDs by clicking the Choose File button. Finally click Assign Membership and the window will report whether or not it is successful on the result line.

To update each of a group’s members configuration files simply locate the group in the list and click the download button in the action column. Note, an application group will use the configuration file associated with each of its individual member’s configuration group.
