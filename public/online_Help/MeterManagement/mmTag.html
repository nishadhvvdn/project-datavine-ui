The tag discrepancy list shows every reported discrepancy between the current configurations on the Meters and what the CollectionENGINE™ is supposed to have set it to.
The list contains five columns.
1. Serial Number: This column shows the serial number of the Meter.
2. Configuration Tag: This column shows the configuration value on the CollectionENGINE™.
3. Discrepant Tag: This column shows the configuration value that is actually on the Meter.
4. First Found Time: The timestamp for when the system first registered the discrepancy.
5.Corrected Time: The timestamp for when discrepancy was corrected (this is blank if it has not been corrected).

If the report has a large amount of data, it might separate into pages. In this case, the user can choose how many entries to display or which page to show from the pagination tool bar in the bottom left of the report.

The filter for the tag discrepancies page contains two sections. The first section allows you to choose between showing tag discrepancies that have already been corrected, ones that havenâ€™t been corrected, and both by clicking the appropriate radio button.

 

The second section allows you to input a date range that will filter the list for discrepancies that were found within the specified range.

 

To sort the list, the user must click the header of whichever column they would like to sort by. For instance, clicking on the Configuration Tag header will switch the column between the default display, ascending alphabetically, and descending alphabetically.

The report gives four different options for exporting. The user may export either all the data in the report or just the currently displayed data in either a CSV format or a PDF format. To do this the user simply has to click on the export button in the top right corner of the report after filtering it for the desired data.

The report can easily be prepared for printing by using the provided printing button. This button will open up a standard printing window with the report displayed.
