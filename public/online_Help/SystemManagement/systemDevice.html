The device management list shows every device that have communicated with the system.
The list contains six columns.
1.Serial Number: This column shows unique serial number of the device.
2.HyperSPROUT™ ID: This column shows unique ID of the device.
3.Name:This column shows the name given to the device.
4.Hardware Version: : This column shows the version number of the hardware on the device.
5.Registered: This column shows whether or not the device has fully registered with the system.
5.Actions: User can perform Device logs, Reboot Device, Factory reset , Lock / Unlock the device from this column..

If the list has a large amount of data, it might separate into pages. In this case, the user can choose how many entries to display or which page to show from the pagination tool bar in the bottom left of the report.

To sort the list, the user must click the header of whichever column they would like to sort by. For instance, clicking on the Group header will switch the column between the default display, ascending alphabetically, and descending alphabetically.

The report gives four different options for exporting. The user may export either all the data in the report or just the currently displayed data in either a CSV format or a PDF format. To do this the user simply has to click on the export button in the top right corner of the report after filtering it for the desired data.

The filter for the device management page contains only one section. This is a checkbox to remove or show non-registered devices from the list.

To search for a device simply click the tab corresponding to the device type user would like to look up (HyperSPROUT™ or meter). Then enter the entire serial number or a portion of it and click search button for pull up matching devices.

To view a deviceâ€™s basic details search for it in the device list and then click its serial number. This will pull up a window with the device details. Note, user can also ping the node from this window.

Reboot Device functionality is applicable for HyperSPROUT™, HyperHUB™, Meter (Mesh Card) and DeltaLINK™. With this functionality the Devices will be remotely Bootable from the Cloud and there is no need to go to the field. User can search Device by serial number or by Device ID from UI and send reset command to the Device.

Lock/Unlock Device functionality is applicable for HyperSPROUT™, HyperHUB™ and Meter. With this functionality, the Device GUI interface (HTML Pages) can be locked from Cloud. If anyone connects to the device then he/she should not see the Device GUI until it is not enabled from cloud.

Delta Device Log functionality is applicable for HyperSPROUT™, HyperHUB™, Meter and DeltaLINK™. With this functionality, the Logs of any Device can be fetched from the Device and stored to the cloud and if any error is device user can check it from the logs. User can enable different types of logs.

Delta Factory Reset Device functionality is applicable for HyperSPROUT™, HyperHUB™, Meter (Mesh Card) and DeltaLINK™, with this functionality user can remotely factory reset any device from the cloud.
