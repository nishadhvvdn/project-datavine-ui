The registration list shows all DTC's,HyperHUBs™,transformers,meters and endpoints (depending on which tab user have clicked) registered to the system.
The list contains different columns depending on which tab user have clicked.
The DTC's list has five columns.
1. DTC ID: The ID of the DTC is shown here.
2. KVA Rating: The KVA rating of the DTC is shown in this column.
3. Sub-Station ID: The ID associated with the substation is located here.
4. Sub-Station Address: The physical address of the substation is in this column.
5. Actions: A user can view, edit, or delete a DTC from this column.
The HyperHUB™ list contains six columns
1. HyperHUB™ Serial Number: This contain HyperHUB™ Serial Number.
2. Hub Name: This contain Hub Name.
3. Hardware Version: This contain Hardware Version of the device.
4. Latitude: The latitude for the transformer’s physical location is in this column.
5. Longitude: The longitude for the transformer’s physical location is in this column.
6. Actions: A user can view, edit, or delete a HyperHUB™ from this column.
The transformer list contains eight columns
1. Transformer ID: The ID of the transformer is shown here.
2. KVA Rating: This column shows the KVA rating of the transformer.
3. HS Serial Number: This column contains the HyperSPROUT™ serial number.
4. HS Name: This one contains the HyperSPROUT's™ name.
5. Hardware Version: The version of the hardware currently on the device is shown here.
6. Latitude: The latitude for the transformer’s physical location is in this column.
7. Longitude: The longitude for the transformer’s physical location is in this column.
8. Actions: A user can view, edit, or delete a transformer from this column.
The meter list contains seven columns.
1. Meter ID: The ID of the meter is shown here.
2. Make: This column shows who makes the meter. For instance, Vision or Tatung.
3. Consumer Number: The consumer ID number is shown in this column.
4. No of Phases: The number of phases supported by the meter.
5. Latitude: The latitude for the meter’s physical location is in this column.
6. Longitude: The longitude for the meter’s physical location is in this column.
7. Actions: A user can view, edit, or delete a meter from this column.
The DeltaLINK™ list has four columns.
1. Serial Number: This column contains the DeltaLINK™ serial number.
2. Version : The version of the hardware currently on the device is shown here.
3. Registered: This column shows whether or not the device has fully registered withthe system.
The endpoint list has five columns.
4. Actions: A user can view, edit, or delete a DeltaLINK™ from this column.
1. MAC ID: This contain MAC ID of the endpoint device.
2. DTC: The ID of the DTC is shown here.
3. Owner: This contain Name of the owner details.
4. Description: This contain detailed information of the endpoint device.
5. Actions: A user can view, edit, or delete a endpoint from this column.

If the report has a large amount of data, it might separate into pages. In this case, the user can choose how many entries to display or which page to show from the pagination tool bar in the bottom left of the report.

To sort the list, the user must click the header of whichever column they would like to sort by. For instance, clicking on the Group header will switch the column between the default display, ascending alphabetically, and descending alphabetically.

The report gives four different options for exporting. The user may export either all the data in the report or just the currently displayed data in either a CSV format or a PDF format. To do this the user simply has to click on the export button in the top right corner of the report after filtering it for the desired data.

To search for a device, first click the tab of the device type user wish to search for. Then enter the entire ID or a portion of it and click search button for pull up matching devices.

There are two options for adding a device. The most simple option is to import the device details from a device specific csv file . To do this click the upload button above the top right corner of the list.Note: User can download device specific CSV file from the window.

After clicking the button a window will pop up. Click the Choose File button to select the file to import and the click the Import button. The device will then be added to the list.

The second option is to manually enter a device’s details. To do this click the Add button directly next to the Upload button. This will open a new window where user can input all of the details manually.

To edit a device simply find it on the list and click the edit button in the Actions column for the device.

This will open a new window where user can edit every detail of the device. After the changes have been made just click the Update button.

To view a device’s details locate in the device list and click the view button in the Actions column for the device.

This will open a new window with the device’s details. Click the OK button when user are done to return to the previous screen.

To delete a device locate in the list and click on the delete button in the Actions column for the device. User will be prompted if user are sure. To delete the device click Yes.

Configuration Management helps the user to configure the device configurations which are present in the device local UI. After registration if user change the configuration from the Cloud then it will reflect in the device. It will be applicable for the HyperSPROUT™ /HyperHUB™/ Meter. When user add a device, it will create automatic entries for the device configuration in the DB. Device configuration is only applicable for HH/HS/Meter (Mesh Card). These configurations will be sent to the device at the time of registration.

With the Non-technical loss calculation, User can identify any theft of electricity happening. It is based upon the difference between actual power delivered and consumption of the Meters or technical loss items (like street light, traffic lights) which is directly consuming the electricity from transformer with minor line loss considered. If the difference is Zero then there is no theft but if the difference is there then user see theft is happening as the consumption doesn’t match the power delivered.
