The firmware version report allows a user to view all firmware update files known to the system and the versions associated with the devices. The CollectionENGINE™ should hold several versions of firmware for a given device class. For example, you might have a version of the firmware just for testing, or an update that you have not implemented yet. However, only one version can be in use at a time. This report displays all of the versions available as well as the version in production. The report contains six columns.
1. Serial Number: The serial number of the device.
2.Hardware Version: The version of the hardware device.
3.HyperSPROUT™: The serial number for the HyperSPROUT™ that the meter communicates with.
4.Firmware Type: The type of firmware installed on the device.
5.Firmware Version: The firmware version installed and controlling the device.
6.Production Version: The firmware version designated for deployment. This may or may not be the firmware currently installed on the device.

If the report has a large amount of data, it might separate into pages. In this case, the user can choose how many entries to display or which page to show from the pagination tool bar in the bottom left of the report.

To sort the report, the user must click the header of whichever column they would like to sort by. For instance, clicking on the Firmware Version header will switch the column between the default display, ascending versions, and descending versions.

The report gives four different options for exporting. The user may export either all the data in the report or just the currently displayed data in either a CSV format or a PDF format. To do this the user simply has to click on the export button in the top right corner of the report after filtering it for the desired data.
